FileCheck工具

FileCheck 是一个基于文本模式匹配的验证工具，广泛用于LLVM和MLIR的测试中。它通过读取输入文件中的CHECK指令，与程序的实际输出进行比对，从而验证IR变换、优化结果或诊断信息的正确性。

  • 模式匹配：支持正则表达式、变量捕获、重复模式等。
  • 灵活定位：可指定匹配顺序或允许乱序匹配。
  • 集成友好：与Lit测试框架无缝集成，是MLIR单元测试的基石。
指令功能示例
CHECK基本匹配模式// CHECK: %0 = arith.addi
CHECK-LABEL标记测试区块// CHECK-LABEL: func.func @test
CHECK-NOT禁止出现模式// CHECK-NOT: "discard_op"
CHECK-DAG允许乱序匹配// CHECK-DAG: pattern1

Lit测试框架

Lit（LLVM Integrated Tester） 是一个轻量级、可移植的测试运行器，用于执行LLVM和MLIR项目中的测试套件。它通过解析测试文件中的RUN指令，调用相应工具并检查返回结果。

  1. 测试发现：递归扫描目录，识别包含RUN指令的测试文件。
  2. 命令执行：执行RUN行中定义的命令序列（如mlir-opt、mlir-translate）。
  3. 结果验证：结合FileCheck验证输出，或检查退出码。

典型测试文件结构：

// RUN: mlir-opt %s --pass-to-apply | FileCheck %s

func.func @example(%arg0: f32) -> f32 {
  // CHECK: arith.addf
  %0 = arith.addf %arg0, %arg0 : f32
  return %0 : f32
}

MLIR验证器

MLIR验证器 是确保IR合法性与一致性的核心组件。它在编译流程的各个阶段自动运行，检查操作、类型、属性、区域、块等是否符合Dialect定义和MLIR规范。

  • 静态验证：基于操作定义（ODS）自动生成验证逻辑。
  • 动态检查：在转换过程中断言IR状态，及早发现错误。
  • 自定义验证：允许Dialect开发者添加特定约束。

验证层次：

  1. 语法验证：IR格式、术语符匹配。
  2. 语义验证：操作数类型匹配、区域结构、副作用合规。
  3. 转换后验证：确保变换未破坏IR不变性。

图形化IR查看器

图形化IR查看器 将文本形式的MLIR转换为可视化图表，极大提升了复杂IR结构的理解与调试效率。工具如 MLIR Visualizer 或 Jupyter Notebook插件 支持交互式探索。

  • 层次化展示：以节点-边图形式呈现操作、块、区域。
  • 交互操作：展开/折叠区域、高亮数据流、查看属性。
  • 转换对比：可视化展示Pass应用前后的IR变化。

典型工作流：

  1. 导出IR到.dot或.svg格式。
  2. 通过图形工具（如Graphviz）渲染。
  3. 在浏览器或IDE中交互查看。
