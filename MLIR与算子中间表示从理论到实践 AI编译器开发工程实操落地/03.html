📦LLVM与MLIR源码编译

获取源码

  • 使用官方LLVM项目仓库，MLIR作为其子项目存在。
  • 推荐使用Git进行克隆，并切换到稳定的发布分支。
  • 确保磁盘空间充足（建议预留20GB以上）。
git clone https://github.com/llvm/llvm-project.git
cd llvm-project
git checkout release/18.x

编译配置

  • 使用CMake进行配置，推荐Ninja作为生成器以加速构建。
  • 关键配置选项：启用MLIR、启用断言、优化级别。
  • 构建类型通常选择Debug（便于调试）或Release（性能最佳）。
mkdir build && cd build
cmake -G Ninja ../llvm \
-DLLVM_ENABLE_PROJECTS="mlir" \
-DCMAKE_BUILD_TYPE=Release \
-DLLVM_TARGETS_TO_BUILD="host"

⚙️开发环境配置：CMake与Bazel

两种主流的构建系统，适用于不同的项目规模和团队偏好。

特性 CMake Bazel
定位 跨平台构建生成器 快速、可扩展的构建与测试系统
依赖管理 需借助FetchContent或find_package 内置的、声明式的外部依赖管理
构建速度 依赖生成器（Ninja快，Make慢） 高级缓存与并行化，增量构建极快
生态 C++项目事实标准，生态成熟 在AI/大规模项目中流行
推荐场景 传统C++项目，与LLVM生态深度集成 大型MLIR项目，需要沙箱与远程缓存

实践建议： 初学者建议从CMake开始，因其与LLVM/MLIR原生集成，文档丰富。大型团队或追求极致构建性能的项目可评估引入Bazel。

🚀第一个MLIR Pass编写与运行

MLIR Pass是转换和优化MLIR程序的核心单元。下面以编写一个简单的“Hello World” Pass为例。

1. Pass骨架

// HelloPass.cpp
#include "mlir/Pass/Pass.h"
using namespace mlir;

struct HelloPass : public PassWrapper<HelloPass, OperationPass<FuncOp>> {
void runOnOperation() override {
FuncOp op = getOperation();
op->emitRemark() << "Hello, MLIR!";
}
};

2. 注册Pass

// 在Pass管理文件中注册
void registerHelloPass() {
PassRegistration<HelloPass>(
"hello", "A pass that says hello."
);
}

3. 运行Pass

使用MLIR优化工具mlir-opt加载并运行自定义Pass。

# 编译生成mlir-opt
ninja mlir-opt

# 运行Pass
./build/bin/mlir-opt input.mlir \
--hello --verify-diagnostics

📚核心要点总结

  • 环境是基石： 稳定的LLVM/MLIR编译环境是后续所有开发与实验的前提。
  • 构建系统选择： 根据项目阶段与团队习惯，合理选择CMake或Bazel。
  • Pass是核心抽象： 理解Pass的编写、注册与运行机制是深入MLIR的关键一步。
  • 工具链熟悉： 掌握mlir-opt、mlir-translate等基础工具的使用。
  • 迭代验证： 编写Pass后，务必通过实际MLIR程序运行验证其正确性。
