MLIR文本格式概述

MLIR (.mlir) 文件是一种人类可读、可写的中间表示文本格式。它承载了编译器中间表示（IR）的完整信息，是连接算法、硬件与编译优化的桥梁。

  • 可读性：采用类似汇编的语法，但结构更清晰。
  • 层次性：支持操作、区域、模块等多级嵌套。
  • 可扩展性：通过Dialect机制支持自定义操作、类型和属性。
  • 可序列化：完整的IR可以持久化到文本文件，并重新加载。

核心语法结构

MLIR文件由一系列操作（Operation）构成，操作是IR的基本单元。

// 模块（Module）是顶级容器 module { // 函数（FuncOp）是一个包含区域的操 func.func @main(%arg0: tensor<2x3xf32>) -> tensor<2x3xf32> { // 运算操作：addf %0 = arith.addf %arg0, %arg0 : tensor<2x3xf32> // 返回操作 return %0 : tensor<2x3xf32> } }

一个操作通常包括：结果、名称、操作数、属性、类型和可选的区域（Region）。

操作（Operation）定义

操作是执行和转换的基本单位，其通用格式为：

[%result =] dialect.operation_name(operands) {attributes} : type_signature (-> return_types)?
组成部分描述示例
结果可选，以%开头的SSA值%0, %result
操作名方言名前缀.操作名arith.addf, func.call
操作数输入SSA值列表%arg0, %1
属性编译期已知的键值对{lower_bound = 0}
类型签名操作数和结果的类型tensor<2xf32>

属性与类型系统

属性（Attribute）是编译期已知的常量元数据，类型（Type）描述数据的形状与语义。

  • 内置属性：整数、浮点数、字符串、数组、字典等。
  • 方言属性：由特定方言定义，如DenseElementsAttr。
  • 内置类型：index, f32, vector<4xf32>。
  • 方言类型：tensor, memref<2x2xf32>。
// 属性示例 %cst = arith.constant 42 : i32 %tensor = arith.constant dense<[[1.0, 2.0], [3.0, 4.0]]> : tensor<2x2xf32> "func.call"(%arg0) { callee = @foo, call_conv = #llvm.cc<"cdecl"> } : (tensor) -> () // 类型示例 %0:2 = "some.op"(%arg0) : (tensor<10xf32>) -> (i64, vector<4xf32>)

区域（Region）与块（Block）

区域是包含一系列块（Block）的容器，块是包含操作列表并具有参数列表的基本执行单元。

func.func @region_example(%arg0: i32) -> i32 { // 这是一个块（Block），参数为 %arg0 ^bb0(%block_arg: i32): %0 = arith.addi %block_arg, %block_arg : i32 // 区域可以嵌套操作，形成控制流图 scf.if %cond { // 新区域（then区域） %1 = arith.muli %0, %0 : i32 scf.yield %1 : i32 } else { // 另一个区域（else区域） scf.yield %0 : i32 } return %result : i32 }

区域使得MLIR能够自然地表示结构化控制流（如循环、条件分支）和复合操作。
