高层次优化（HLO）在MLIR中的实现

计算化简、常量折叠、代数简化与死代码消除的核心原理与工程实践

高层次优化（HLO）概述

在AI编译器中，高层次优化（High-Level Optimization, HLO）是在计算图或算子中间表示（IR）的较高抽象级别上进行的转换与简化。其目标是在不改变程序语义的前提下，提升执行效率、减少资源占用并简化后续优化。

MLIR（Multi-Level IR）通过其可扩展的方言系统和模式重写框架，为HLO的实现提供了强大且统一的底层支持。优化以模式（Pattern）的形式定义，并由驱动程序（Driver）在IR上应用。

四大核心优化技术详解

1. 计算化简

将复杂的计算表达式替换为更简单、等价的表达式。例如，将 x * 1 简化为 x。

  • 目标：减少计算操作数量。
  • MLIR实现：通过 CanonicalizationPattern 注册化简规则。

2. 常量折叠

在编译时计算仅涉及常量的表达式，并用计算结果替换。

  • 目标：消除运行时计算开销。
  • MLIR实现：实现 OpFoldResult 接口或使用 fold 方法。

3. 代数简化

利用代数恒等式（交换律、结合律、分配律等）重组表达式。

  • 目标：暴露更多优化机会，如公共子表达式消除。
  • MLIR实现：定义 RewritePattern 应用代数定律。

4. 死代码消除

识别并删除对程序输出无影响的代码（指令、变量、函数）。

  • 目标：减少代码体积和内存占用。
  • MLIR实现：基于使用-定义链分析，使用 eraseOp 删除无用操作。

MLIR中的实现模式对比

优化类型 关键MLIR组件/接口 应用时机 工程收益
计算化简 CanonicalizationPattern, getCanonicalizationPatterns 构建期、优化期 减少算子数量，简化图结构
常量折叠 Op::fold, OpFoldResult, 常量属性 编译时（常量传播后） 直接降低运行时计算负载
代数简化 RewritePattern (DRR), 属性跟踪 优化期（循环、融合前后） 提升数据局部性，利于融合
死代码消除 SideEffect, MemoryEffect, eraseOp 优化后期、代码生成前 减少内存占用与执行指令

实例：常量折叠与代数简化

优化前MLIR片段（Toy方言示例）:

%c1 = arith.constant 1.0 : f32 %c2 = arith.constant 2.0 : f32 %sum = arith.addf %c1, %c2 : f32 // 常量折叠目标 %prod = arith.mulf %sum, %c1 : f32 // 代数简化目标 (乘以1) %result = toy.transpose(%prod) : (f32) -> f32

应用HLO优化Pass后:

%c3 = arith.constant 3.0 : f32 // 常量折叠：1.0 + 2.0 = 3.0 %result = toy.transpose(%c3) : (f32) -> f32 // 代数简化：删除 %prod = %sum * 1

通过组合应用常量折叠和代数简化，两条计算指令被消除，直接使用编译期计算出的常量结果。

工程实践要点

  1. 模式注册顺序：简单的模式（如常量折叠）应优先于复杂的模式（如代数简化），以创造更多优化机会。
  2. 副作用分析：死代码消除必须准确识别操作的副作用（SideEffect），避免删除必要的I/O或内存操作。
  3. 自定义方言集成：为自定义MLIR方言实现 Op::fold 和 getCanonicalizationPatterns 以接入通用HLO框架。
  4. 测试驱动：使用MLIR的 FileCheck 工具为每个优化模式编写单元测试，确保语义正确性。
  5. 组合优化：将HLO Pass编排到优化管道（Pass Pipeline）中，与循环变换、数据布局优化等协同工作。
