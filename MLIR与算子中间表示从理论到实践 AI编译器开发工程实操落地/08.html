MLIR模式匹配与重写

深入解析声明式重写规则(DRR)、C++重写模式与贪婪重写器，构建高效可靠的编译器转换基础设施

核心概念

在MLIR中，模式匹配与重写是编译器转换的核心机制，它允许开发者定义如何将一种IR模式转换为另一种更优化或更规范的IR模式。这一过程是MLIR多层次中间表示能够进行渐进式 lowering 和优化的基础。

MLIR提供了多种方式来表达重写规则，从高级的声明式语法到灵活的C++ API，适应不同复杂度和性能需求的转换场景。

声明式重写规则 (DRR)

使用基于TableGen的声明式语法，简洁定义源模式到目标模式的映射。

  • 高可读性：类似DSL的语法，意图清晰
  • 易于维护：规则集中管理，减少样板代码
  • 自动验证：MLIR自动检查模式合法性
  • 快速原型：快速实现常见规范化转换

C++重写模式

通过C++ API实现，提供最大灵活性和控制力，处理复杂转换逻辑。

  • 完全控制：访问所有IR信息和基础设施
  • 复杂逻辑：支持条件转换、副作用处理
  • 动态匹配：运行时决定匹配和重写行为
  • 性能优化：精细控制匹配和重写过程

贪婪重写器

驱动重写过程的基础设施，应用模式直到达到不动点(fixed point)。

  • 迭代应用：重复应用模式直到无变化
  • 冲突解决：处理模式间的优先级和冲突
  • 工作列表算法：高效管理待处理操作
  • 可配置策略：控制重写顺序和范围

架构与工作流程

MLIR重写系统架构

贪婪重写器作为驱动引擎，从工作列表中选取操作，通过模式匹配引擎查找适用的重写规则（包括DRR和C++模式），应用转换后更新工作列表，迭代直到达到不动点。

最佳实践与工程建议

规则设计原则

  1. 单一职责：每个规则只完成一个明确的转换
  2. 幂等性：规则多次应用结果应一致
  3. 局部性：尽量基于局部信息做决策
  4. 渐进式：复杂转换分解为多个简单规则

性能优化策略

  1. 模式优先级：为高频规则设置更高优先级
  2. 早期剪枝：在匹配早期排除不可能情况
  3. 批量重写：合并相关重写减少迭代次数
  4. 缓存结果：缓存昂贵计算或匹配结果
工程经验：在实际编译器开发中，通常混合使用DRR和C++模式。DRR用于处理大量简单、规范的转换，保持代码简洁；C++模式处理需要复杂逻辑或特殊处理的转换。贪婪重写器的配置（如最大迭代次数、区域限制）对编译时间和转换结果有重要影响，需要根据具体场景调整。

应用场景示例

代数化简

使用DRR实现常量折叠、恒等变换等代数化简规则：

  • add(x, 0) → x
  • mul(x, 1) → x
  • add(const1, const2) → const3
  • and(x, x) → x

规范化

将操作转换为规范形式，便于后续优化：

  • 统一操作数顺序
  • 消除冗余转换
  • 标准化类型表示
  • 扁平化嵌套结构

Lowering准备

为跨层次lowering做准备：

  • 分解复合操作
  • 插入显式转换
  • 调整内存布局
  • 处理平台特定约束
